## Design Context

### Users
Primary users are power users and maintainers of Claude Code Router who run the tool locally or self-host it for their own coding workflow. They need to connect providers, assign routing lanes, inspect fallback posture, and tune advanced runtime behavior without dropping into raw JSON for every change.

### Brand Personality
Technical, transparent, self-hosted. The interface should feel like a control plane rather than a marketing dashboard: calm, capable, and explicit about what the router will do. Emotional goal: confidence through visibility, not through decoration.

### Aesthetic Direction
Manifest is the reference, specifically its strengths in control-plane information architecture, routing explainability, and tier-based configuration. The UI should stay local-first and operational rather than trying to imitate Manifest's product surface literally. Prefer warm light surfaces, dense but legible cards, and clear sectioning over generic app-shell layouts.

### Design Principles
1. Show routing policy before raw configuration details.
2. Make every route understandable as a lane with a primary target and a fallback path.
3. Keep advanced tools nearby but secondary to the common workflow.
4. Preserve local-first, self-hosted credibility with explicit operational status and config integrity.
5. Prefer explainability and configuration safety over visual novelty.
